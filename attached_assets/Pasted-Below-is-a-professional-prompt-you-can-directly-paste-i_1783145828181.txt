# ERP Workflow Update

Implement a complete synchronized workflow between the following modules:

* Quotations
* Orders
* Invoices
* Customers

The document hierarchy must always be:

**Quotation → Order → Invoice**

Every document must know its parent.

Example:

* Order stores quotation_id
* Invoice stores order_id
* Invoice also stores quotation_id if applicable

The database must always maintain proper relationships.

# Quotation Module

Quotation statuses:

* Draft
* Sent
* Accepted
* Rejected

---

## Draft

When quotation is Draft:

* Can Edit
* Can Delete
* No Order should exist
* No Invoice should exist
* No PDF should exist

---

## Sent

When status changes

Draft

↓

Sent

Automatically:

* Generate Quotation PDF
* Download PDF automatically
* Save PDF reference
* Keep quotation editable
* No Order should be created yet

If quotation was previously saved as Draft and later changed to Sent, PDF must be generated at that moment.

---

## Accepted

When quotation changes

Sent

↓

Accepted

Automatically:

* Create Order
* Link Order with quotation_id
* Remove "Convert to Order" action button
* Keep View PDF
* Keep Download PDF
* Keep Edit
* Keep Delete

Delete should still be available even after acceptance.

---

## Rejected

When quotation becomes Rejected:

* No Order should exist
* No Invoice should exist

---

# Editing Rules

Editing a Quotation should update related Order and Invoice information where applicable.

# Database Rules

Maintain proper parent-child relationships.

Use foreign keys or references where applicable.

Never leave orphan Orders or Invoices.

Always validate parent existence before creating child records.

Use database transactions so that if one operation fails, all related operations roll back to maintain data consistency.
